Schema

  1. 사용자(User) 테이블:

    • 테이블 이름: users
    • 열:
      • UserID (INT): 고유한 사용자 식별자 (기본키)
      • UserName (VARCHAR): 사용자 이름
      • Email (VARCHAR): 이메일 주소
      • PasswordHash (VARCHAR): 비밀번호 해시
      • CreatedAt (TIMESTAMP): 계정 생성 일시
      • UpdatedAt (TIMESTAMP): 계정 정보 업데이트 일시

  2. 자세 점수표(PostureScore) 테이블:

    • 테이블 이름: posture_scores
    • 열:
      • ScoreID (INT): 고유한 점수표 식별자 (기본키)
      • UserID (INT): 사용자 식별자 (외래 키)
      • Date (DATE): 자세 기록 날짜
      • Score (INT): 자세 점수

  3. 통계 데이터(Statistics) 테이블:

    • 테이블 이름: statistics
    • 열:
      • StatID (INT): 고유한 통계 데이터 식별자 (기본키)
      • UserID (INT): 사용자 식별자 (외래 키)
      • StatDate (DATE): 통계 데이터 날짜
      • AverageScore (DECIMAL): 해당 날짜의 평균 자세 점수
      • MaxScore (INT): 해당 날짜의 최고 자세 점수
      • MinScore (INT): 해당 날짜의 최저 자세 점수

API

  1. 사용자 등록 (User Registration)

    • 엔드포인트: POST /api/users/register
    • 요청 본문 (Request Body): 
      {
  "userName": "john_doe",
  "email": "john@example.com",
  "password": "secure_password"
}
    • 응답 (Response): 
      {
  "message": "User registered successfully"
}

  2. 로그인 (User Login)

    • 엔드포인트: POST /api/users/login
    • 요청 본문 (Request Body): 
      {
  "email": "john@example.com",
  "password": "secure_password"
}
    • 응답 (Response): 
      {
  "message": "Login successful",
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwiaWF0IjoxNjEwMjM5MDIyfQ.XXXXXX"
}

  3. 매일 자세 점수 저장 (Save Daily Posture Score)

    • 엔드포인트: POST /api/scores
    • 요청 본문 (Request Body): 
      {
  "userID": 123,
  "date": "2023-09-20",
  "score": 95
}
    • 응답 (Response): 
      {
  "message": "Posture score saved successfully",
  "scoreID": 456
}

  4. 특정 사용자의 자세 점수 조회 (Get User's Daily Posture Scores)

    • 엔드포인트: GET /api/scores/user/{userID}
    • 요청 매개변수 (Query Parameter):
      • 날짜 범위 (옵션): startDate와 endDate로 날짜 범위 지정 가능
    • 응답 (Response): 
      {
  "scores": [
    {
      "scoreID": 1,
      "date": "2023-09-18",
      "score": 92
    },
    {
      "scoreID": 2,
      "date": "2023-09-19",
      "score": 88
    },
    {
      "scoreID": 3,
      "date": "2023-09-20",
      "score": 95
    }
  ]
}

  5. 통계 데이터 생성 및 조회 (Generate and Get Statistics Data)

    • 엔드포인트: GET /api/statistics/user/{userID}
    • 요청 매개변수 (Query Parameter):
      • 통계 유형 (옵션): type=daily (일간 통계), type=weekly (주간 통계), type=monthly (월간 통계) 등 통계 유형 선택 가능
    • 응답 (Response): 
      {
  "statistics": [
    {
      "statID": 101,
      "statDate": "2023-09-18",
      "averageScore": 90.5,
      "maxScore": 92,
      "minScore": 89
    },
    {
      "statID": 102,
      "statDate": "2023-09-19",
      "averageScore": 88.2,
      "maxScore": 90,
      "minScore": 86
    },
    {
      "statID": 103,
      "statDate": "2023-09-20",
      "averageScore": 91.1,
      "maxScore": 95,
      "minScore": 89
    }
  ]
}

  6. 로그아웃 (User Logout)

    • 엔드포인트: POST /api/users/logout
    • 요청 헤더 (Request Header): Authorization: Bearer <access_token>
    • 응답 (Response): 
      {
  "message": "Logout successful"
}

  7. 비밀번호 재설정 (Reset Password)

    • 엔드포인트: POST /api/users/reset-password
    • 요청 본문 (Request Body): 
      {
  "email": "john@example.com"
}
    • 응답 (Response): 
      {
  "message": "Password reset link sent successfully"
}
